# Benchmarks

Use this directory for benchmark scripts that are broader than a single toy project.

Examples:

- comparing implementations across multiple projects
- checking interpreter or compiler effects
- scaling studies that deserve their own script
- storing notes or small result snapshots worth keeping

Rule of thumb:

- If the timing code only exists to support one project, keep it inside that project.
- If the benchmark compares ideas across projects or has independent value, put it here.

Keep benchmark scripts lightweight and runnable from the repo root.
